# Reviews

This directory is reserved for local Phase 2.27a report review records.

Runtime review files are intentionally ignored:

- `reviews/**/*.json`
- `reviews/**/*.md`

Do not commit real review records by default. They may contain reviewer names, business risk notes, fact ids, document ids, report hashes, and follow-up decisions.

Templates or schema examples may be committed under a dedicated examples directory if needed, but real review output should stay local or move to a controlled artifact store.
