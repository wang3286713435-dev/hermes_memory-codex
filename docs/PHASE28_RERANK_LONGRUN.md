# Phase 2.8 更长期小流量运行与运行期观测

## 1. 阶段目标

本阶段目标不是扩大启用范围，而是验证：

1. 当前局部默认启用运行基线在更长期的小流量运行中是否持续稳定
2. 规则层是否继续保持稳定
3. 运行层是否继续可控、可观测、可回退
4. 是否已经足以支撑下一步更高一级 rollout 评估

## 2. Phase 2.8 退出标准

本阶段收口不再依赖开放式长期观察，而是依赖一个固定的最后观察窗口。当前退出标准如下：

### A. 运行基线稳定

需要同时满足：

1. 最后观察窗口内命中集不漂移
2. 最后观察窗口内误命中保持为 `0`
3. 最后观察窗口内无新增漏命中边界样本
4. 最后观察窗口内正常窗口 `fail_open_rate` 持续接近 `0`

### B. 运行期开关与回退稳定

需要同时满足：

1. `global_disabled` 正常
2. `local_disabled` 正常
3. `timeout_drill` 正常
4. fail-open 行为与 trace 一致、可诊断

### C. 延迟可接受

当前阶段采用基于既有观察区间的保守工程标准：

1. 最后观察窗口内正常窗口 `p95 <= 350 ms`
2. 最后观察窗口内正常窗口 `p99 <= 520 ms`
3. 不出现继续恶化的长尾趋势

说明：

- `p99` 允许存在单次尖峰，但不应继续抬升到明显高于既有观察区间
- 若连续窗口 `p99` 持续高于 `520 ms`，则当前阶段不能收口

### D. 后端异常窗口可控

当前阶段采用现实可执行标准：

1. 最后观察窗口内不应再次出现 `dense_failed_count > 0`
2. 最后观察窗口内不应再次出现 `sparse_failed_count > 0`
3. 若再次出现并导致目标命中集整体归零，则当前阶段不能收口
4. 历史上允许存在一次已恢复的孤立异常窗口，但最后观察窗口内不应复现

## 3. 最后观察窗口

最后观察窗口固定为 `3` 次完整观察，继续沿用：

- 同一条脚本：`scripts/phase26_rerank_gray_validation.py`
- 同一套规则
- 同一套启用范围
- 同一套统计口径

## 4. 当前保持不变的运行基线

当前运行基线保持不变：

- `RERANK_PROVIDER=aliyun`
- `ALIYUN_RERANK_MODEL=gte-rerank-v2`
- `ALIYUN_RERANK_TIMEOUT_MS=600`
- `RERANK_ENABLED=true`
- `RERANK_DEFAULT_ENABLEMENT_ENABLED=true`

启用条件保持不变：

1. request 侧 `source_type / route_type` 命中
2. query 命中关键词
3. candidate count 达到阈值

当前启用范围保持不变：

1. 招标资料相关 query
2. 资质要求类
3. 条款定位类
4. 时间安排 / 商务条款 / 评分标准类高歧义招标 query

## 6. 逐条判定

### A. 是否满足运行基线稳定标准

满足。

理由：

1. 命中集在最后观察窗口内保持同一组 `13` 条目标 query
2. 误命中保持为 `0`
3. 未出现新的漏命中边界样本
4. 正常窗口 `fail_open_rate` 持续为 `0`

### B. 是否满足运行期开关与回退稳定标准

满足。

理由：

1. `global_disabled` 正常
2. `local_disabled` 正常
3. `timeout_drill` 正常
4. fail-open 行为与 trace 一致且可诊断

### C. 是否满足延迟标准

满足。

理由：

1. 最后观察窗口内 `p95 = 279.198 / 259.384 / 299.722 ms`，均低于 `350 ms`
2. 最后观察窗口内 `p99 = 291.179 / 510.078 / 361.528 ms`，仍在当前阶段允许区间内
3. 未观察到持续恶化趋势

### D. 是否满足后端异常窗口可控标准

满足。

理由：

1. 最后观察窗口内 `dense_failed_count > 0` 未复现
2. 最后观察窗口内 `sparse_failed_count > 0` 未复现
3. 未出现目标命中集整体归零

## 7. 分层结论

### A. 规则层

当前规则层继续保持稳定：

1. 命中集不漂移
2. 误命中保持为 `0`
3. 未出现新的漏命中边界样本

### B. 运行层

当前运行层在本轮窗口内是稳定的：

1. 未再出现新的后端瞬时失败窗口
2. `fail_open_rate` 保持接近 `0`
3. 全局关闭 / 局部关闭 / timeout drill 继续有效

但运行层仍需继续注意：

1. `p99` 仍存在长尾尖峰
2. 历史上出现过 `1` 次已恢复的后端异常窗口

## 8. 当前阶段结论

当前局部默认启用策略已经能够作为一套稳定、可控、可观测、可回退的运行基线，在更长期的小流量运行中持续成立。

但当前仍应保持：

1. 不扩大启用范围
2. 不进入全局默认启用
3. 不改当前规则

## 9. Phase 2.8 收口判断

当前可以收口。

原因：

1. 最后观察窗口满足运行基线稳定标准
2. 最后观察窗口满足运行期开关与回退稳定标准
3. 最后观察窗口满足当前阶段定义的延迟标准
4. 最后观察窗口内未再次出现后端异常窗口

下一阶段建议命名为：

- `Phase 2.9：局部默认启用运行期基线维持与更高一级 rollout 评估准备`

说明：

Phase 2.8 的收口不意味着扩大启用范围，也不意味着进入更高一级 rollout 实施。下一阶段应以维持当前运行基线、补运行期观测沉淀和 rollout 评估准备为主。
