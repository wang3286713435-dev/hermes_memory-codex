# Codex A Output Protocol

本文件定义 Codex A 在本项目中的长期输出规范。

除非用户明确取消或要求改用其他格式，否则后续所有回复默认按本协议执行。

## 1. 固定输出字段

后续所有回复必须严格按以下字段输出，字段名保持不变：

1. 本轮目标
2. 修改文件
3. 本轮完成内容
4. 测试与验证结果
5. 当前结论
6. 阻塞点 / 风险点
7. 下一步建议
8. 是否建议进入下一阶段
9. 文档同步

不得随意改字段名、删字段、增加自由发挥的大段总结。

## 2. 压缩规则

1. 不复述已知背景；除非背景发生变化，不重复解释旧 bug、阶段背景或长段铺垫。
2. 每个字段只写必要信息；优先短句或短点。
3. 同一判断在同一回复里只表达一次，不重复换说法。
4. `测试与验证结果` 必须优先使用：
   - 已完成：
   - 部分完成：
   - 未完成：
5. `当前结论` 必须直接写核心判断，例如：
   - 当前阶段是否收口：是 / 否
   - 当前 bug 是否已解除：是 / 否
   - 是否恢复到可继续推进当前阶段：是 / 否
6. `下一步建议` 只允许写唯一优先事项，或最多两项优先事项。
7. `修改文件` 只列路径，不解释。
8. `文档同步` 只写：
   - 已更新哪些文档
   - 未更新哪些文档（如有）
   - 是否已写入关键结论

## 3. 长度要求

1. 普通回复尽量控制在 25 行以内。
2. 复杂回复尽量控制在 40 行以内。
3. 不因简短而漏字段，也不因完整而写成长篇汇报。

## 4. 适用范围

本协议适用于：

1. 所有阶段任务回复
2. 所有 bug 修复回复
3. 所有环境修复回复
4. 所有阶段收口回复
5. 所有 readiness / gray / real-file / retrieval 相关回复

## 5. 优先级

如果后续上下文压缩，本协议仍优先保留并继续执行。
