# MVP Tender Review Playbook

## 1. 适用范围

用于内部试运行阶段的标书审查 / 风险提取。

目标是辅助人工快速定位条款，不是自动审标或自动决策。

## 2. 输出格式

建议每次输出固定为：

1. 结论。
2. 证据来源。
3. 风险等级：高 / 中 / 低 / 未判定。
4. 待人工确认项。
5. 未找到 evidence 的事项。

## 3. 基础信息提取模板

```text
围绕 @主标书，提取工程名称、工程地点、建设单位、代建单位、工期、最高投标限价。
请按字段输出：
- 字段名
- 结论
- evidence / citation
- 是否需要人工确认
如果当前召回中没有找到，请明确写“未在当前召回中找到”。
```

## 4. 投标资格提取模板

```text
围绕 @主标书，提取投标资格要求：
资质等级、项目经理要求、联合体要求、业绩要求、人员要求。
请输出结论、证据来源、风险等级、待人工确认项。
不得用无 evidence 的推断补齐。
```

## 5. 商务 / 合同风险模板

```text
围绕 @主标书，提取商务和合同风险：
付款、质保、工期、违约、结算、变更、索赔。
请按风险项输出：
- 风险项
- 条款摘要
- evidence / citation
- 风险等级
- 待人工确认项
```

## 6. 缺失信息识别模板

```text
围绕 @主标书，检查是否能找到【指定事项】。
如果没有 retrieval evidence，请输出“未在当前召回中找到”，不要凭经验补充。
```

## 7. 硬规则

1. 所有结论必须有 evidence / citation。
2. 无 evidence 时必须明确缺失。
3. confirmed facts 不得替代标书 evidence。
4. 会议纪要不得作为招标文件条款引用。
5. 历史记忆不得替代当前 retrieval evidence。
