# Phase 2.9 小批量脱敏企业资料受控验证计划

## 1. 当前为什么可以开始尝试企业资料验证

当前已经具备以下前提：

1. 局部默认启用运行基线已固定
2. 启用范围已固定
3. 当前规则与关键词集合已固定
4. runbook、long-run 和 readiness 文档已完成
5. 双仓库可回滚 baseline 已完成

因此，当前可以开始尝试更接近真实业务场景的资料适配性验证。  
但这个阶段的目标不是扩大全量知识库，而是验证：

1. 当前 retrieval / rerank 主链路对更真实资料是否仍保持稳定
2. 当前局部启用范围在真实资料上是否仍保持低误命中
3. rollout readiness 是否能积累更贴近业务的数据支撑

## 2. 当前为什么必须先从脱敏 / 低敏 / 小批量开始

当前必须先从受控样本开始，原因如下：

1. 当前 Air 仍不是高可信长窗口最终裁决机
2. 当前 Phase 2.9 的重点仍是 readiness 维持，而不是大规模数据接入
3. 若直接接入全量真实企业资料，会把数据风险、权限风险和系统验证风险混在一起
4. 小批量、脱敏、低敏样本更容易隔离问题来源，也更容易在出现异常时快速回退

## 3. 当前允许接入的资料类型

当前允许接入以下资料类型：

1. 脱敏招标文档样本
2. 脱敏制度文件样本
3. 低敏公司文档样本
4. 公开材料
5. 已明确授权范围内的样本资料

这些资料应满足：

1. 可用于 retrieval / rerank 主链路验证
2. 不包含高敏个人信息
3. 不包含未脱敏客户信息
4. 不包含真实密钥、账号、预算或未公开商业决策

## 4. 当前不应接入的资料类型

当前不应接入以下资料类型：

1. 未脱敏高敏合同
2. 含客户隐私信息的资料
3. 含账号、token、密钥、访问凭证的资料
4. 含内部预算、未公开商业决策、敏感定价策略的数据
5. 权限范围未确认的内部资料
6. 高敏或全量企业知识库直接接入

## 5. 当前验证方式与边界

当前企业资料验证方式应保持为：

1. 小批量
2. 单独验证集
3. 受控授权范围内
4. 不改变当前规则
5. 不改变当前启用范围
6. 不影响主线开发节奏

更具体地说：

1. 资料验证只作为 Phase 2.9 的补充证据来源
2. 不把企业资料验证和 rollout 实施绑定
3. 不把企业资料验证结果外推出更广 query 类型

## 6. 与 Phase 2.9 主线如何衔接

当前这项资料验证与 Phase 2.9 的关系是：

1. 为 readiness 提供更接近真实业务的辅助证据
2. 验证当前局部启用范围在更真实资料上的适配性
3. 帮助区分“规则问题”和“资料适配问题”
4. 不改变当前 rollout readiness 的主线判断逻辑

换句话说：

1. Phase 2.9 主线仍然是 readiness 维持与 rollout 评估准备
2. 企业资料验证只是受控补充，不是当前主线替代品

## 7. 当前不做什么

当前明确不做：

1. 全量企业资料接入
2. 高敏资料直接接入
3. 因企业资料验证结果而立即改规则
4. 因小样本验证结果而扩大启用范围
5. 把当前资料试验直接当作 rollout 实施依据

## 8. 当前阶段结论

当前更稳妥的工程策略是：

1. 白天继续主线开发
2. 晚间继续长窗口验证
3. 同步开始小批量、脱敏、低敏企业资料受控验证
4. 所有资料验证结果都应服务于 Phase 2.9 的 readiness 累积
5. 在没有更高可信验证环境之前，不把这类试验直接上升为 rollout 结论
